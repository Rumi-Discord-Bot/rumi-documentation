<Info>
コンテキストが重要です。変数が返す `N/A` 現在のコマンドまたはイベントがソース値を提供しない場合。追加 `|fallback text` よりフレンドリーなフォールバックが必要な場合は、右中括弧の前に置きます。
</Info>


| 変数 | 返されるもの | 出力例 | で利用可能 |
| --- | --- | --- | --- |
| <code>&#123;if:&lt;left&gt;&lt;operator&gt;&lt;right&gt;&#125;yes&#123;else&#125;no&#123;endif&#125;</code> | ==、!=、を使用して値を比較します。 &gt;, &lt;, &gt;=、または &lt;=. | <code>カスタム コマンド ガイドの構文と例を参照してください。</code> | カスタムコマンド。カスタムデータヘルパーには自動応答の --db も必要です |
| <code>&#123;if:&lt;value&gt; contains &lt;text&gt;&#125;yes&#123;else&#125;no&#123;endif&#125;</code> | 大文字と小文字を区別せずに、テキストに別の値が含まれているかどうかを確認します。 | <code>カスタム コマンド ガイドの構文と例を参照してください。</code> | カスタムコマンド。カスタムデータヘルパーには自動応答の --db も必要です |
| <code>&#123;if:exists &lt;value&gt;&#125;yes&#123;else&#125;no&#123;endif&#125;</code> | 解決された値が存在し、 false でないかどうかを確認します。 | <code>カスタム コマンド ガイドの構文と例を参照してください。</code> | カスタムコマンド。カスタムデータヘルパーには自動応答の --db も必要です |
| <code>&#123;if user has permission &lt;permission&gt;&#125;yes&#123;else&#125;no&#123;endif&#125;</code> | チェックしてください Discord ユーザーまたはターゲットに対する権限。 | <code>カスタム コマンド ガイドの構文と例を参照してください。</code> | カスタムコマンド。カスタムデータヘルパーには自動応答の --db も必要です |
| <code>&#123;if user has role @Role&#125;Allowed&#123;else&#125;Denied&#123;endif&#125;</code> | メンバーに役割があるかどうかを確認します。 | <code>許可された</code> | カスタムコマンド、自動応答 |
| <code>&#123;if:&#123;user.id&#125;==&#123;target.id&#125;&#125;No self target.&#123;else&#125;OK&#123;endif&#125;</code> | 安全な条件が合格または失敗した場合に、別のテキストを表示します。 | <code>わかりました</code> | カスタムコマンド、自動応答 |
