## Rumi 返事をしない

1.確認 Rumi オンラインです。
2. チャンネルを表示し、メッセージを送信できることを確認します。
3. プレフィックスとコマンドのスペルを確認します。
4. 試してみる `/help` または `-help`.
5. コマンドがサーバーまたはダイレクト メッセージに制限されているかどうかを確認します。

## スラッシュコマンドがありません

Discord グローバルに登録されたコマンドを更新するには時間がかかる場合があります。再起動 Discord,
ボットのアプリケーション コマンド権限を確認し、プレフィックス フォームを試してください。

## 言語は変更されませんでした

走る `-language` 効果的な言語とそのソースを表示します。次に、
意図したスコープを明示的に指定します:

```text
-language user de
-language server de
```

個人言語はサーバー言語より優先されます。サーバー全体の変更
管理者の許可が必要です。

## Rumi サーバーを離れました

Rumi リソースを保護するために、サーバーは 30 日間アクティビティが記録されないままにしておきます。
そしてサービスの安定性。招待できます Rumi またいつでも。メンバーが使用すると、
サーバーと Rumi アクティビティを記録すると、通常の非アクティブ期間が再び始まります。

## 管理コマンドが拒否されました

走る `-help <command>` メンバーとボットの両方の要件を確認してください。役割について
またはチャンネル変更、 Discordの役割階層は引き続き適用されます。 Rumi 管理できない
最高の役割以上の役割。

## 禁止またはキックが見つかりません

Rumi 意図的に汎用を提供していません `ban` または `kick` によるコマンド
デフォルト。これにより、ベースラインの権限要件が小さく抑えられます。サーバ
管理者は、慎重に制限されたワークフローを構築できます。
[カスタムコマンドシステム](../custom-commands/overview).

## Spotify コントロールは再リンクを要求します

プレーヤー、デバイス、プレイリスト、および「いいね！」した曲の機能​​は異なるものを使用します。 Spotify
権限。走る `-spotify link` 再度、更新された承認を承認します。
これらの機能が追加される前に作成された既存のリンクには、新しいアクセス権が付与されません。
自動的に。

Spotifyの個人用デバイスのコントロールには、アクティブな Spotify デバイスと Spotify
プレミアム。開ける Spotify 目的のデバイスで、一度再生を開始してから、次のコマンドを実行します。

```text

```

## レベルの役割が割り当てられていません

3 つの条件をすべて確認してください。

1. Rumi **役割の管理** があります。
2. Rumiの最高の役割は報酬の役割を上回ります。
3. マイルストーンが表示されます。 `-levels settings`.

設定を修正した後、実行します `-levels sync` を使用して報酬の役割を調整するには
現在の構成。

## 招待参加は不明です

リフレッシュ Rumiの現在の招待スナップショット:

```text
-invites refresh
```

Rumi 招待が削除された場合、バニティ リンクが削除された場合、帰属を保証することはできません。
使用されているか、招待アクセスが欠落している、または Discord 十分な状態を提供できませんでした
どのカウンタが変更されたかを特定します。スナップショットを更新すると、今後の結合に役立ちますが、
古いものを再構築しません。

## メディア変換が失敗する

入力値を小さくするか短くして、直接添付ファイルまたはファイル URL を使用してください。のために
ビデオ変換、縮小 `--duration`; GIF 出力の場合、期間を短縮し、フレームを短縮します
レート、または幅。プラットフォームのダウンロードはメディア サービスに依存しており、たとえ
添付ファイルが機能する場合。

## カスタムコマンドが出力されます `N/A`

変数は存在しますが、その実行ではそのソース値が存在しませんでした。ターゲット
たとえば、変数には解決可能なターゲットが必要です。フォールバックを追加します。

```text
{target.display_name|No member selected}
```

使用 `-cc show <name>` 保存されたスクリプトと `-cc プレビューを検査するには <name>
[サンプル引数]` を使用して安全に再現します。 [テストガイド]](../custom-commands/testing-and-debugging)
各レイヤーを見ていきます。

## カスタムアクションが拒否されました

保護されたアクションは、構成メンバー、カスタムを実行しているメンバーをチェックします。
指示、 Rumiの権限、ターゲット、ロール階層、およびコマンド自体
制限。レビュー：

```text
-cc show <name>
-cc restrictions <name>
-permissions @member
```

アクションを成功させるためだけにすべての制限を削除しないでください。特定の部分を修正する
代わりに、ロール、チャネル、権限、または階層の問題が考えられます。

問題が解決しない場合は、
[サポートサーバー](https://discord.gg/NYs2jF9MCc).
