どのレイヤーが失敗したかがわかっている場合、カスタム コマンドをデバッグするのが最も簡単です。テストしてください
最初に引数、次に変数、次に埋め込み、最後に保護されたアクションを追加します。

## 何を検査するか Rumi 保存されました

```text
-cc show <name>
```

結果には、保存された引数の仕様、応答スクリプト、および現在の値が表示されます。
制限。応答が間違った場所から始まる場合は、スタンドアロンを確認してください。
`--` 原作で使われている `cc set` 指示。

どの名前が使用されたかわからない場合は、保存されたすべてのコマンドをリストします。

```text
-cc list
```

## 現実的な引数を使用してプレビューする

```text
-cc preview greet @member
-cc preview announce Saturday 8pm
```

プレビューでは、指定したサンプル引数を使用して保存されたテンプレートが実行されます。を使用します。
メンバーが実際のコマンドで使用する引数と同じ数とタイプ。

引数の問題の場合は、プレーン テキストへの応答を一時的に減らします。

```text
-cc set announce <day> <time> -- Day: {day}
Time: {time}
```

これらの値が正しい場合は、埋め込みまたはアクションを復元します。

## 欠落している変数を診断する

変数は、現在のコンテキストで利用可能な情報のみを返すことができます。のために
たとえば、ターゲット変数にはターゲット引数または解決されたメンバーが必要です。追加
値がオプションのフォールバック:

```text
{target.display_name|No member selected}
```

生成された [変数リファレンス] を使用します。](../variables/overview) または実行します:

```text
-variables target
-variables message
-variables customdb
```

変数がリテラル テキストとして表示される場合は、そのスペルと中括弧を確認してください。もしそうなら
返品 `N/A`、そのソース値がそのコマンド コンテキストに存在しませんでした。

## 条件を個別にテストする

Rumi 存在チェック、比較、テキストの包含、役割、および
Discord 権限:

```text
{if:{user.id} == 123456789012345678}
This is the configured user.
{else}
This is somebody else.
{endif}
```

他の条件形式には次のものがあります。

```text
{if:{message.content} contains hello}
{if:user has role 123456789012345678}
{if:user has permission ManageMessages}
{if:{db.get:visits:{user.id}} >= 10}
```

開口部を維持する `{if:...}`、オプション `{else}`、そして終了 `{endif}` 彼らの
デバッグ中に独自の行を作成します。条件はネストできますが、フラットな条件は
通常はメンテナンスが容易になります。

## 表示とアクションを分離する

まず無害な応答を保存します。

```text
-cc set staffwarn <target> <reason...> --
Would warn {target.mention} for: {reason}
```

安全なターゲットを使用してプレビューします。その後でのみアクションを追加します。

```text
{mod:warn target && {reason}}
Warned {target.mention} for: {reason}
```

最後に、ロール、チャネル、または権限の制限を追加します。 Rumi 検証する
保存時に保護されたアクションが実行され、コマンドの実行時にチェックが繰り返されます。

## コンポーネントの問題

ボタンまたはドロップダウンが表示されない場合:

1. コマンドをプレビューして読みます Rumiの検証応答。
2. リンク ボタンの URL 形式を確認します。
3. 各行のコンポーネントと合計行を数えます。
4. ラベル、オプションの説明を短くするか、コンテンツを埋め込みます。
5. アクション ボタンに不足しているターゲットまたは権限が必要かどうかを確認します。

Discord メッセージは最大 5 つのコンポーネント行をサポートします。 Rumi 埋め込みも強制します。
ペイロードを送信する前に、フィールド、ボタン、およびドロップダウンの制限を設定します。

## 保存されたデータの問題

名前空間とキーを直接検査します。

```text
-cc data list visits
-cc data get visits 123456789012345678
```

名前空間とキー名は別のものです。 `visits:user-id` 同じ検索ではありません
として `user-id:visits`.

<Tip>
大規模な編集を行う前に作業コピーを保存してください。 `-cc show <name>` 正確な情報を提供します
保存された回答は、置き換える前にメモにコピーできます。
</Tip>
