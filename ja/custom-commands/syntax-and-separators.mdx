＃＃ なぜ `--` ありますか

セパレータは次のように伝えます Rumi コマンド定義が終了する場所と保存される場所
応答が始まります:

```text
-cc set greet <member> [message...] -- Hello {member}! {message|Welcome.}
```

Rumi 行を 3 つのセクションに分けて読み取ります。

| セクション | 価値 |
| --- | --- |
| 手術 | `-cc set` |
| 意味 | `greet <member> [message...]` |
| 保存された応答 | `Hello {member}! {message\|Welcome.}` |

区切り文字がないと、応答内の単語が引数のように見える可能性があります
宣言。区切り記号を使用すると、その曖昧さが解消されます。その間消費されます。
コマンドは保存され、応答に含まれることはありません。

`=>` そして `->` 代替手段として受け入れられていますが、 `--` ドキュメント全体で使用されています
長いスクリプトで認識するのが最も簡単だからです。

<Warning>
最初のスタンドアロン `--`, `=>`、 または `->` が境界です。それ以降は何でも
応答テキストが次で始まる場合でも、 `--`。引数の宣言を前に置く
境界線。
</Warning>

## 宣言された引数

| 形状 | 意味 |
| --- | --- |
| `<member>` | 必須の 1 単語の引数 |
| `[reason]` | オプションの 1 語の引数 |
| `<text...>` | 必須の残りの引数。残ったものをすべて消費する |
| `[note...]` | オプションの残りの引数 |

引数名は文字で始まり、文字、数字、 `_`, `-`,
または `.`。 REST 引数は残りの入力をすべて消費するため、最後に置く必要があります。

## 複数行のスクリプト

Discord メッセージには改行を含めることができます。ディレクティブを独自の行に配置します。
それらは表示される応答からきれいに削除されます。

```text
-cc set daily --
{cooldown:24h}
{customdb:increment rewards && {user.id} && 1}
You claimed reward number **{db.get:rewards:{user.id}}**.
```

ランタイムでサポートされる他のスタンドアロン ディレクティブには次のものがあります。 `{delete}` に
呼び出し元のメッセージを削除し、 `{silent}` アクションのみのコマンドの場合、
`{respond:#channel}` 応答チャネルを選択するには、 `{dm:user}` を送信する
結果はDMにて、 `{reaction:emoji}` トリガーに反応するため。

## 値とフォールバック

使用 `{variable|fallback}` 値が存在しない可能性がある場合:

```text
Nickname: {user.display_name|No server nickname}
Target: {target.mention|No target selected}
```

Rumi 宣言された引数を最初に解決し、次にヘルパー値と条件を解決します。
次に、コンテキスト変数と最終的な埋め込み/コンポーネント スクリプトです。
