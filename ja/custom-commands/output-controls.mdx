出力コントロールは、カスタム コマンド応答内のスタンドアロンのディレクティブ行です。
彼らはやり方を変える Rumi は結果を送信しますが、返信テキストとしては表示されません。

## トリガーを削除します

```text
{delete}
```

Rumi カスタム コマンドを実行したメッセージを削除します。これには**管理が必要です
ソースチャネル内のメッセージ**。削除に失敗した場合は、コマンドの残りの部分を実行できます。
他の設定に従って処理されます。

## サイレント出力

```text
{silent}
```

構成されたアクションまたはデータのみを実行するコマンドにはサイレント モードを使用します。
更新するため、通常の目に見える応答を送信すべきではありません。開発中に、
アクションがテストされるまで、目に見える確認を残しておきます。

## 応答をルーティングする

```text
{respond:#suggestions}
```

Rumi レンダリングされた出力を選択したサーバー チャネルに送信します。それはできるはずです
そのチャネルを表示し、コマンドで使用されるペイロード タイプを送信します。

例：

```text
-cc set report <text...> --
{respond:#reports}
$v{embed}
$v{author: {user.display_name} && {user.avatar}}
$v{description: {text}}
$v{footer: Sent from {channel.name}}
```

レポートを次の場所からのみ送信する必要がある場合は、コマンドのソース チャネルを制限します。
特定の場所。

## ダイレクトメッセージを送信する

```text
{dm:user}
```

出力は、パブリック チャネルではなく、選択したユーザー ターゲットに送信されます。
受信者がサーバーメンバーからの DM をブロックしている場合、または受信者がサーバーメンバーからの DM をブロックしている場合、DM 配信は失敗することがあります。
ブロックされました Rumi.

## リアクションを追加する

```text
{react:✅}
{react:✅ && ❌}
```

Rumi を読み取ることができると、リストされた反応をトリガー メッセージに追加します。
メッセージを送信し、それらの反応を追加します。カスタムサーバー絵文字は、次のユーザーでも使用できる必要があります。
Rumi.

## ユーザーごとのクールダウン

```text
{cooldown:10m}
```

クールダウンは、メンバーによるカスタム コマンドの繰り返し使用に適用されます。使用
読み取り可能な期間など `30s`, `10m`、 または `24h`.

```text
-cc set daily-note --
{cooldown:24h}
You can use this response again tomorrow.
```

## コントロールを慎重に組み合わせる

```text
-cc set confidential <text...> --
{cooldown:5m}
{delete}
{dm:user}
Your saved note: {text}
```

追加する前にルーティングと DM をテストする `{delete}`。それ以外の場合は、宛先が失敗します
トリガーメッセージも期待される応答も得られない可能性があります。

<Warning>
出力制御は、権限ではなく配信を変更します。内部で保護されたアクション
同じコマンドが引き続き実行されます Rumiの権限、制限、対象、役割
階層チェック。
</Warning>
