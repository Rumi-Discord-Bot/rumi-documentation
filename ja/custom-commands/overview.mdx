カスタム コマンドを使用すると、サーバー管理者は追加することなく詳細なワークフローを作成できます。
汎用的なモデレーション コマンドの大規模なセット。テキストを生成したり、
埋め込み、引数の受け入れ、変数の解決、ボタンの表示、保護された実行
アクション、制限の適用、およびサーバー データの保存。

主なコマンドは `customcommand`。その別名は次のとおりです。 `cc` そして
`customcommands`.

<Info>
カスタム コマンドを作成および変更するには、サーバー管理権限が必要です。
現在の無料制限はサーバーあたり 30 コマンドです。対象となるプレミアムサーバーは、
最大200個まで保存できます。
</Info>

## コマンドマップ

| タスク | 指示 |
| --- | --- |
| 作成または編集 | `-cc set <name> [arguments] -- <response>` |
| テスト | `-cc preview <name> [sample arguments]` |
| 検査する | `-cc show <name>` |
| リスト | `-cc list` |
| 消去 | `-cc remove <name>` |
| 変数を表示する | `-cc variables` または `-variables` |
| アクションの構文を表示する | `-cc actions` |
| 制限を管理する | `-cc restrictions <name>` |
| 保存したデータを管理する | `-cc data <operation> ...` |

## 推奨されるワークフロー

<Steps>
  <Step title="最小限の有用な応答を作成する">
    プレーン テキストと 1 つまたは 2 つの引数から始めます。
  </Step>
  <Step title="プレビューしてみよう">
    走る `-cc preview <name>` リアルなサンプル入力を使用します。
  </Step>
  <Step title="アクセスを制限する">
    保護された使用の前に、ロール、チャネル、ユーザー、または権限の制限を追加します。
  </Step>
  <Step title="保存されたコマンドを検査する">
    走る `-cc show <name>` その対応と制限事項を確認します。
  </Step>
</Steps>

<Warning>
コマンドをロール、メッセージ、エコノミー、またはメンバー管理アクションで扱います。
スタッフの道具。信頼できるマネージャーのみが編集できるようにする必要があります。
</Warning>

## 保存されるもの

各コマンドは、その応答テンプレート、宣言された引数、およびアクセスを保存します。
制限。ランニング `set` 同じコマンドを再度更新します。保存されたワークフロー
データは分離されているため、コマンドを置換または削除しても、暗黙的に削除されるわけではありません。
そのカウンターまたはレコード。
