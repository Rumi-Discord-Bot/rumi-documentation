Elke server heeft zijn eigen server Rumi economie. Valutanamen, beloningsbedragen, instellingen,
saldi en winkelitems worden bewaard in de economie van die server in plaats van gedeeld
tussen niet-verwante gemeenschappen.

## Wat leden kunnen doen

De dagelijkse opdrachten zijn eenvoudig:

```text
-balance
-balance @member
-daily
-weekly
-work
-deposit 500
-deposit all
-withdraw 250
-give @member 100
-economytop
```

Kas- en banksaldi zijn gescheiden. `deposit` brengt contant geld naar de bank;
`withdraw` verplaatst het terug. Overboekingen gemaakt met `give` kan worden beïnvloed door de
de door de server geconfigureerde overdrachtsbelasting.

## Initiële configuratie

Leden met **Manage Server** kunnen de instellingen bekijken en bewerken:

```text
-economy config
-economy config currency petals
-economy config icon 🌸
-economy config daily 250
-economy config weekly 2500
-economy config work 50 350
-economy config cooldown work 3600
```

Het werkbereik is een minimale en maximale uitbetaling. Cooldowns zijn ingevoerd
seconden in het configuratiecommando voor zuinigheid.

Schakel een besparingsopdracht uit zonder het hele systeem uit te schakelen:

```text
-economy config disable rob
-economy config enable rob
```

`economy config reset confirm` reset de instellingen naar hun standaardwaarden, maar doet dat wel
saldi niet verwijderen. `economy reset confirm` is anders: het reset de
gebruikerssaldi van de server. Lezen Rumi's bevestiging zorgvuldig voordat u een van beide gebruikt.

## Overvallen en casinospellen

Robben kan worden afgestemd of uitgeschakeld:

```text
-economy rob
-economy rob on
-economy rob cooldown 3600
-economy rob limits 25 500
-economy rob chance 35
-economy rob fine 20
-economy rob protection 12
```

Leden gebruiken `-rob @member [amount]`. Rumi hanteert de uitgespaarde kans, grenzen,
boete voor mislukte pogingen, afkoelperiode en beschermingsperiode.

Casinocontroles werken op dezelfde manier:

```text
-economy casino on
-economy casino cooldown 20
-economy casino limits 10 1000
-casino coinflip 100 heads
-casino slots 100
```

## Winkels en inventarissen

Leden kunnen door de serverwinkel bladeren, items kopen, hun inventaris inspecteren en
ondersteunde items verkopen:

```text
-shop
-shop view <item>
-buy <item>
-inventory
-sell <item>
```

### Bouw een winkelitem

Leden met **Manage Server** kunnen serverwinkelitems maken en onderhouden. De
formaat maken is `name`, `buy price`, optioneel `sell price`, Dan `|` en de
beschrijving:

```text
-shop add coffee 150 45 | Fresh server coffee
-shop add vip-pass 5000 0 | Unlocks the VIP role
-shop view coffee
```

Itemnamen zijn enkele tokens, dus gebruik `vip-pass` in plaats van `VIP pass`. Een verkoop
prijs van `0` voorkomt dat het artikel geld retourneert wanneer het wordt verkocht. `edit` gebruikt
hetzelfde volledige formaat:

```text
-shop edit coffee 175 50 | A warm drink and a small energy boost
```

Gebruik voor kleinere wijzigingen de gerichte opdrachten:

```text
-shop price coffee 200
-shop sellprice coffee 60
-shop desc coffee A limited seasonal drink
-shop remove coffee
```

### Door rollen ondersteunde items

Voeg er één of meerdere toe Discord rollen voor een item:

```text
-shop role add vip-pass @VIP
-shop role add vip-pass @Supporter
-shop role remove vip-pass @Supporter
```

Rumi moet in staat zijn elke daaraan verbonden rol, dat wil zeggen de hoogste rol, te beheren
moet boven hen zitten Discord's rollenlijst.

### Artikelcodes en cadeaubonnen

Bij een artikel wordt een artikelcode opgeslagen die door het personeel kan worden onderhouden:

```text
-shop code create vip-pass WELCOME1
-shop code remove vip-pass WELCOME1
```

Cadeaubonnen zijn inwisselbare valutacodes die aan een artikel zijn gekoppeld. Lever de
valutabedrag en aantal gebruiken na de code:

```text
-shop giftcard create vip-pass LAUNCH500 500 25
-shop giftcard remove vip-pass LAUNCH500
```

Leden verzilveren een geldige code met:

```text
-shop redeem LAUNCH500
```

Elke succesvolle inwisseling voegt het geconfigureerde bedrag toe aan dat van dat lid
winst en verbruikt één gebruik. Behandel cadeauboncodes als privélinks: iedereen
wie een geldige code kent, kan proberen deze in te wisselen zolang er nog gebruik van kan worden gemaakt.

`rumishop` is Rumi's wereldwijde botwinkel en staat los van de server-geconfigureerd
winkel.

## Wijzigingen controleren

Rumi registreert veranderingen in de economie-instellingen en transacties:

```text
-economy audit
-economy logs
```

Deze zijn handig na het wijzigen van beloningen, belastingen, overvalinstellingen of casinolimieten.

<Tip>
Stel eerst conservatieve beloningen en inzetlimieten in. Het is gemakkelijker om een
het tempo van de economie dan het herstellen van de balansen na een al te genereuze opzet.
</Tip>
